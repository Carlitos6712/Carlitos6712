## 🧱 How I write code

| Principle | Practice |
|---|---|
| **SOLID** | Single responsibility, open/closed and dependency inversion by default |
| **Clean Code** | Functions under 30 lines — names that read like documentation |
| **REST standards** | JSON `{success, data, message}` · correct HTTP verbs always |
| **Security first** | No secrets in code — env vars, centralised error handling, no empty catch |
| **Performance** | Eager loading to avoid N+1 — query optimisation before premature scaling |
| **Documentation** | JSDoc · PyDoc · PHPDoc · JavaDoc — comment the *why*, not the *what* |
